MySekai Planner — UI wireframes / low-fi · 5 directions · pick a vibe

Goal: take the current makeshift editor and re-skin it to feel like the in-game MySekai layout tool — soft mint panels, cream cards, chunky rounded corners. Below: a quick design-system note, then 5 distinct structural directions on the same canvas. Pick one (or mix), and we commit to a polished design pass next.

palette ↳ pulled from in-game

mint · cream · blush · violet · ink · grass

shape language

corners ≈ 18–28px · panels float w/ 2–3px ink stroke · cards have soft drop shadow · 4–6px chunky button shadow

type

UI: rounded sans (M PLUS Rounded) · numbers tabular · ja/zh: Noto Sans JP/SC

A. Floating panels

in-game matchspacious

All chrome floats over a full-bleed canvas, just like the actual game UI. Tool well + catalog drawer on the left, cost panel + save cluster on the right, page dots bottom-center.

Looks closest to the real game, breathing room around the canvas.

Catalog and cost can each collapse to icon-only when not in use.

Floating panels can occlude the corners of the design — needs auto-hide on hover-out.

Less canvas width on small screens.

B. Fixed sidebars (polished current)

least riskystructure-preserving

Same skeleton as today — toolbar / left catalog / center canvas / right cost panel — re-skinned with the in-game palette. Cost panel grows: progress bars per material, layout-cost meter, preset slots.

Smallest engineering change; existing components map 1:1.

Material costs always visible — strong for the planning use case.

Doesn't read as in-game-y; feels more "tool".

Eats canvas width with two fixed columns.

C. Bottom hotbar + drawers

mobile-firstmost canvas

Catalog becomes a wide bottom strip (like a hotbar/inventory). Tools rail left, tiny cost handle right that expands into a panel on tap. Maximum canvas width.

Familiar from mobile games; thumb-friendly drag direction.

Most canvas real estate of any direction here.

Cost panel is hidden by default — extra tap to see materials.

Long category lists need horizontal scroll, less scannable than a vertical grid.

D. Top pill-tabs + radial cost

most novelcompact

All categories live as a horizontal pill bar at the top — fewer clicks to jump categories. Cost becomes a radial dial in the corner: a glance shows which materials are biggest cost.

Top pill bar is fast — one tap to category, no nested menus.

Radial cost is a unique signature element that's compact + scannable.

Radial labels get cramped if there are >8 materials.

Top bar can feel busy with all categories visible at once.

E. Diorama frame

most game-likecohesive

The whole app sits inside a soft mint "room", canvas is a cream-framed window. Side panels are stacked "books" with cream caps. Strongest visual identity, most tied to in-game vibes.

Reads as a finished product, not a tool. Distinct, memorable.

Stacked "books" let us fit cost + presets + actions on the right without one giant scroll.

Frame eats screen real estate (esp. on smaller windows).

Mint-everywhere can feel monotone — needs careful accent rhythm.

F. A + D · floating, with top pill-tabs

A's bonesD's category bar

A's full-bleed floating layout, but categories live as a pill-tab strip up top instead of inside the catalog drawer. Catalog stays expanded by default (PC has the room).

Switching categories = single tap, no nested chrome.

Catalog drawer becomes a clean grid with no internal tabs — better item density.

Top bar gets crowded if there are >10 categories — may need overflow ▸.

G. E + D · diorama with top pill-tabs

most distinctcohesive

The diorama frame, but the catalog "book" no longer holds category tabs — those move to the pill bar at top, freeing the catalog to be a clean item grid. Most game-like + fastest category access.

Strongest visual identity of the bunch.

Categories at the very top, items on the side — clear hierarchy.

Top bar floats on mint, side panels float on mint — needs careful contrast tuning.

H. A + E · floating "books" over canvas

A's canvasE's panel chrome

A's full-bleed canvas, but every floating panel becomes an E-style "book" with a cream cap header. Side panels feel like polished objects sitting in the world, not flat HUD chrome.

Cream caps add visual rhythm — each panel reads as its own thing.

Keeps maximum canvas real estate (no mint-room overhead like G).

More chrome density on the sides than plain A.

No top pill-tabs — categories still nested inside catalog book.

A–E are the original directions; F–H are remixes; F2 drills into the chosen direction with cleaner organization, collapse states, and category↔catalog connector variations

F2 · F deep-dive

Cleaning up the toolbar mess from F. New rule: verbs of the same kind cluster together — actions (save/share/undo) on the top-right, modes (select/place/erase/rotate) on the left tool rail, canvas state (zoom/preset slot/grid) along the bottom. The 9 dots = preset slots (assumed). Then four ways to visually connect the top pill-tab to the catalog underneath.

F2-a · cleaned-up baseline + collapsed catalog

two states side by side: catalog open (default) ↔ collapsed (canvas takes over)

F2-b · how the category pill connects to the catalog

four ways. select the active pill = "全部"; catalog reflects it.

recommendation

pick ② color echo + ① anchor tail together — color makes the link readable at a glance, the tail confirms it on inspection. Both are subtle in static mockup but feel obvious in motion.

avoid ④ pills-on-catalog if you ever want category switching while catalog is collapsed (you'd lose the category bar entirely). Good if collapse is rare; bad if it's a primary state.

Toolbar reorganization (top of pane): home + project on far left; categories in the middle; undo / redo / share / save on the right. Modes (select/place/erase + rotate/fence/road) drop into a vertical rail next to the catalog, where they should have been. The 1–9 strip at bottom is now a hotbar (drag furniture in, press 1–9 to hot-swap), with level/size/zoom on the same row.
