## Known Issues (Phase 02.1)

### 1. Fence cursor offset (snapping limitation)
**Source:** Plan 03 — `pointerToGrid` uses `Math.floor` snapping to tile origin
**Symptom:** Fence appears at top-left of cursor instead of under it
**Root cause:** Edge painting should snap to nearest grid edge, not tile corner
**Severity:** UX annoyance — functional but unintuitive
**Fix scope:** Improve `pointerToGrid` to support edge-aware snapping

### 2. Fence erase mode unreliable
**Source:** Plan 03 — erase path matches edges by exact floor-snapped `(x, y)`
**Symptom:** Remove mode feels inconsistent; edges slightly off snapping point are missed
**Root cause:** Same floor-snap issue + erase matching ignores edge proximity
**Severity:** Functional bug — erase works sometimes but misses often
**Fix scope:** Implement nearest-edge erase matching (align with cursor offset fix)

### 3. No drag-from-catalog-to-canvas
**Source:** Not implemented (by design)
**Symptom:** Users expect to drag items from sidebar onto canvas
**Severity:** Feature gap — not a bug
**Fix scope:** New feature, separate phase
