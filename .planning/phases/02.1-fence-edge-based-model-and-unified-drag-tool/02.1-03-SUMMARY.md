# Phase 02.1 Plan 03: Unified Drag Handler + Axis Lock Summary

**One-liner:** Fence fixtures now paint edges via the same drag handler roads use; first cross-tile sample locks the axis, progressive samples project onto it, occupancy checks apply, and the whole stroke commits as one zundo entry.

## Accomplishments

- **4 new refs** on EditorCanvas: `isEdgePaintingRef`, `edgeStrokeStartRef`, `lastEdgePointRef`, `edgeStrokeAxisRef` — mirror the Phase 2 tile-paint ref pattern
- **paintEdgeStroke helper** (35 lines of body) — reads start/last/axis refs, locks axis on first cross-tile sample via `dominantAxis(dxTotal, dyTotal)`, projects onto locked axis, calls `rasterizeEdgeLine` + `checkCanPlaceEdge` + `placeEdge` with local edgeSet dedup
- **advanceTilePaint / advanceErase helpers** extracted from `handleMouseMove` — byte-identical logic to the Phase 2 inline bodies, but now callable from a slim dispatch
- **handleStageMouseDown** gains `'drag-paint-edge'` branch next to the existing `'drag-paint'` branch — initializes all four edge refs and calls `startStrokeBatch()`
- **handleMouseMove** rewritten as a 35-line dispatcher: read pointer, resolve active fixture once, branch on edge/tile/erase ref guards
- **endStrokeIfActive** gains edge cleanup branch — refs cleared BEFORE `endStrokeBatch()` (R-06 idempotency)
- **eraseGroundTileIfAny** extended to erase edges FIRST (top-layer priority), then ground tile as before

## pointerToGrid field verification

Read Plan 01's output: `pointerToGrid(stage, step)` returns `{ gridX, gridY, rawGridX, rawGridY }` natively. No inline computation needed — `rawGridX/rawGridY` are the integer floor-based lattice indices used throughout the project for non-snapped cursor position. For edges, they double as the lattice point.

## Decisions Made

- **Direct branching over strategy pattern** — research Option A, CLAUDE.md simplicity-first. Three if-branches in handleMouseMove are clearer than a dispatch table for a stateful React component.
- **Extracted tile-paint + erase helpers** (not just edge helper) — the 50-line limit made this necessary. Net effect: three symmetric helpers, all invoked from a small dispatcher.
- **Edge erase uses exact lattice match** (`edge.x === rawGridX && edge.y === rawGridY`) — simplest correct rule; covers the two edges (top + left) of the cursor's current tile. Plan 05 can add a stricter nearest-lattice rule if smoke testing shows ergonomic issues.
- **Shared activeFixture IIFE** in `handleMouseMove` — avoids duplicating the `getState().activeFixtureId → fixtureMap.get()` pattern across the two paint branches.
