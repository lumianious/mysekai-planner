**In `src/utils/edgeRasterize.ts`:**

1. Add a new pure function `getEdgesForTileFootprint(x, y, w, d)` that returns all edge keys bordering a rectangular footprint of tiles. For a furniture item at (x, y) with size (w, d):
   - Top horizontal edges: for col in [x..x+w-1]: edge (col, y, 'h')
   - Bottom horizontal edges: for col in [x..x+w-1]: edge (col, y+d, 'h')
   - Left vertical edges: for row in [y..y+d-1]: edge (x, row, 'v')
   - Right vertical edges: for row in [y..y+d-1]: edge (x+w, row, 'v')
   - Returns `string[]` (array of edgeKey strings)

2. Extend `checkCanPlaceEdge` signature to accept an optional `occupancyGrid?: OccupancyGrid` parameter (import OccupancyGrid type from editorStore). When provided, additionally check: the two tiles adjacent to this edge must not be occupied in the occupancyGrid.
   - For h-edge (x, y, 'h'): check tiles (x, y) and (x, y-1)
   - For v-edge (x, y, 'v'): check tiles (x, y) and (x-1, y)
   - Use `tileKey` from grid.ts for lookup. Skip tiles outside grid bounds (y-1 < 0 or x-1 < 0 are just boundary edges, only check the valid tile).
   - When occupancyGrid is undefined, behavior is unchanged (backward compatible).

**In `src/stores/editorStore.ts`:**

   - When edgeSet is undefined, behavior is unchanged (backward compatible).
