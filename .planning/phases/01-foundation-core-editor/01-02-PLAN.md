    1. Create `src/stores/editorStore.ts`:

    Use `create` from zustand wrapped in `temporal` from zundo.

    Store shape follows the EditorState interface from types/editor.ts. Key implementation details:

    **Initial state:**
    - areaLevel: 1, gridSize: { width: 36, depth: 36 }
    - placedItems: {}, selectedItemId: null, toolMode: 'select'
    - activeFixtureId: null, overwriteEnabled: false
    - previewRotation: 0 as Rotation (preview rotation for stamp mode ghost preview — lives in store so GhostPreview, useEditorActions, and useKeyboard can all access it without prop-drilling)
    - hotbar: Array(9).fill({ fixtureId: null })
    - isEditorReady: false

    **Actions:**
    - `placeItem(item)`: Generate id with `crypto.randomUUID()`, add to placedItems record
    - `moveItem(id, x, y)`: Update x, y of item by id
    - `rotateItem(id, direction)`: Cycle rotation. 'cw': (current + 90) % 360, 'ccw': (current + 270) % 360
    - `removeItem(id)`: If `placedItems[id]?.isSystem === true`, return (no-op). Otherwise delete from placedItems, clear selectedItemId
    - `setToolMode(mode)`: Set toolMode, clear selectedItemId when switching
    - `setAreaLevel(level)`: Use `getGridSize(level)` from areaLevels.ts. Set both areaLevel and gridSize
    - `setActiveFixture(fixtureId)`: Set activeFixtureId, switch toolMode to 'stamp' if fixtureId is not null, **reset previewRotation to 0** (new fixture starts at default rotation)
    - `setSelectedItem(id)`: Set selectedItemId
    - `toggleOverwrite()`: Flip overwriteEnabled boolean
    - `assignHotbar(slot, fixtureId)`: slot is 1-9 (user-facing), store at index slot-1
    - `activateHotbar(slot)`: Read fixtureId from hotbar[slot-1]. If not null, set activeFixtureId and toolMode to 'stamp'
    - `startEditor(level)`: Set areaLevel, gridSize, isEditorReady = true, auto-place mandatory fixtures (gate and house)
    - `rotatePreview(direction: 'cw' | 'ccw')`: Cycle previewRotation. 'cw': (current + 90) % 360, 'ccw': (current + 270) % 360. Used by keyboard handler (R key in stamp mode) and consumed by GhostPreview and handleCanvasClick for placement rotation.

    **zundo temporal config:**
    ```typescript
    {
      limit: 50,
      partialize: (state) => ({ placedItems: state.placedItems }),
    }
    ```

    Note: `previewRotation` is NOT tracked by zundo (it's transient UI state, not part of the undo/redo history).

    **Occupancy grid helper** (not in store, exported separately):
    ```typescript
    export type OccupancyGrid = Map<string, string>; // tileKey -> itemId

    export function buildOccupancyGrid(
      items: Record<string, PlacedItem>,
      fixtureMap: Map<number, Fixture>,
      layer: ItemLayer
    ): OccupancyGrid;

    export function checkCanPlace(
      occupancyGrid: OccupancyGrid,
      x: number, y: number,
      width: number, depth: number,
      gridWidth: number, gridDepth: number,
      excludeItemId?: string
    ): boolean;
    ```

    - `buildOccupancyGrid`: Iterate items, filter by layer, compute effective size per rotation, populate map with tileKey -> itemId for each occupied tile
    - `checkCanPlace`: Check isInBounds first, then check each tile in the footprint against the occupancy grid. If excludeItemId provided, skip that item (for move validation)
